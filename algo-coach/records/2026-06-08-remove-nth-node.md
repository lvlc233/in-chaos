# 删除链表的倒数第 N 个结点 (Remove Nth Node From End of List)

LeetCode 19 | 归档: 2026-06-08 | 状态: 待复习

## 追问阶段卡点

1. **两趟扫描的惯性**：最初思路是先数长度再定位，被提醒题目要求一趟扫描
2. **间距计算**：n=3 时设间距 2（即 n-1），导致慢指针落在目标节点本身——单链表删节点需要从前驱操作，更正为间距 n+1 或引入哑节点
3. **哑节点的价值**：不仅解决前驱定位问题，还能统一处理删除头节点的边界

## 核心洞察

快慢指针的本质是**制造固定间距**——像一把竹竿，竿头触底时竿尾自然指向倒数第 N 位。间距设为 n+1（或哑节点等效）让慢指针落在前驱位置。

## 双指针模式的扩展认知

双指针不仅是"两端向中间靠拢"（如 11、15），也可以是**固定绝对距离的并行指针**（如本题快慢指针）。后者的关键是：间距不变，利用"一端触底"来定位另一端。
