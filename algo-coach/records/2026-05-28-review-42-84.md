# 复盘：接雨水 (42) & 最大矩形 (84) & 不可逆操作符

日期: 2026-05-28

## 涵盖算法
- 42. 接雨水 (已阅读)
- 84. 柱状图中最大的矩形 (已阅读)

## 核心洞察（一句话）
加法可遗忘（有逆元），min 不可遗忘（无逆元）——不可逆的操作符需要静态化、分段或显式多窗口维护。

## 追问阶段卡点与突破

1. **分层扫描的退化** → 突破：max_h 无界，分层浪费在空层上。单调栈是事件驱动，只关心有变化的位置。

2. **双指针为什么安全** → 突破：局部决定全局——较矮的一边已确定瓶颈，不需要知道另一边精确多高。

3. **个人思路遗漏反例 [3,6,5]** → 突破：Kadane 型贪心只维护一个窗口，min 有记忆，最早短板会"污染"后续所有计算。需要显式切换起点或同时维护多个窗口。

4. **min 不可逆的根源** → 突破：从群论第一性原理推导——加法群有单位元 0（可达、可用、可重置），min 的单位元是 ∞（不可达）。加法可以"回零"，min 无法"回无穷"。

## 新概念：不可逆操作符的六种处理策略

| 策略 | 做法 | 适用 | 不适用 |
|------|------|------|--------|
| 静态化 | 预计算查表 | 条件可独立定义 | 边界动态变化 |
| 方向分解 | 正反各扫一次 | 双边独立约束 | 交叉依赖 |
| 多窗口维护 | 单调栈存待定候选 | 边界触发单调 | 非单调触发 |
| 分段 | 分治/按块切 | 有天然分割点 | 瓶颈跨段 |
| 代数变换 | 换到可逆群 | 存在同态映射 | min/max 无此映射 |
| 显式切换 | 手动重置起点 | 切换条件简单 | 旧窗口仍需维护 |

核心判断：瓶颈信息有没有方向性。

## 两道题的结构对比

| | 42. 接雨水 | 84. 最大矩形 |
|---|---|---|
| 目标 | 求凹槽里的水 | 求凸块面积 |
| 单调栈方向 | 递减（等上升填坑） | 递增（等下降截断） |
| DP 绕开方式 | 双指针模糊另一方 | 预计算 left/right 边界 |
| 空间最优 | O(1)（双指针） | O(n)（必须存边界） |
| 无上位替代 | 是，场景不同 | 是，场景不同 |

## 关联
- DP 中 Kadane (sum) vs 最大矩形 (min)：操作符的代数结构决定问题复杂度
- 加法的 0（单位元）可达 → 可贪心；min 的 ∞ 不可达 → 不可贪心
- 以后遇到 max/min 为主操作符的问题，默认警惕——大概率不可贪心
