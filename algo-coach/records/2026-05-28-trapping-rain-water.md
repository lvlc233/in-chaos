# 42. 接雨水

日期: 2026-05-28 | 状态: 已阅读（未整理）

## 核心洞察

瓶颈约束问题：每个位置的水量 = `min(左最高墙, 右最高墙) - 自身高度`。`min` 是两个独立维度的合取，短板上限决定结果。

## 三种解法

| 解法 | 时间复杂度 | 空间 | 视角 | 计算单元 |
|------|-----------|------|------|---------|
| 双指针 | O(n) | O(1) | 按列 | 每列水量 = min(l_max, r_max) - h[i] |
| 单调栈 | O(n) | O(n) | 按坑 | 每次出栈算一个凹槽 = 宽 × 深 |
| 分层扫描 | O(n × max_h) | O(1) | 按层 | 每层找最左最右柱子，中间计数 |

## 卡点与突破

- 双指针为什么 O(n)：较矮的一边已确定瓶颈，局部决定全局（单调性）
- 单调栈 vs 分层：分层扫描浪费在空层上，单调栈只关心有变化的位置（事件驱动压缩）
- 无上位替代：三者 trade-off 不同——双指针极致效率但信息量最少，单调栈语义丰富，分层直觉最强
- max_h 敏感的算法在 max_h 无界时退化是致命问题

## 结构性质分析

### 单调消去规则 → 双指针

**性质**：较矮的一边已经到达上限，移动它才能打破瓶颈，移动高的那边不可能更优。

```
height[left] < height[right] → left 是瓶颈
→ 以 left 为一边的所有组合里，right 固定在最右是最优的
→ left++ 排除这一边，搜索空间减半
→ 每一步消掉一条边，O(n)
```

**判定条件**：目标函数 = `瓶颈 × 宽度`，瓶颈被 `min` 控制，`min` 的两个参数一个固定（已选边）、一个可变（另一边），且"固定边较矮 → 已是最优"成立。

### 瓶颈单调触发 → 单调栈

**性质**：新元素触发一批待定项，触发条件单调——新元素比栈顶大（或小）时一次性释放。

```
遇到更高的 → 之前的低谷形成凹槽 → pop 计算水
遇到更矮的 → 之前的高柱被截断 → pop 计算矩形面积
```

每个元素进一次出一次，O(n)。栈维护"还没找到右边界"的候选。

### 两性质在同一题的关系

接雨水(42)同时具备两个性质：
- 双指针视角：每个位置的水量由 min(左,右) 决定
- 单调栈视角：凹槽在遇到更高柱子时被触发

两个都是 O(n)，但空间 O(1) vs O(n)。双指针信息量更少（只要总量），单调栈信息量更多（每个坑独立）。
