# in-chaos

一个在混沌中意外开启的 skill 包。每个 skill 都不知道怎么开始写的，但意外地好用。

## Skills

### [iteration-in-chaos](./iteration-in-chaos/)
把用户反馈转化为可复用的迭代技术体系。状态驱动的三层架构——技术（检查方法）、偏好（编排选择）、日志（原则提取）。
- 核心输出：可复用的迭代方案 (techs/)
- 状态机设计：7 状态 + transitions，任何状态都是合法入口
- 自迭代：用自身流程迭代自身

### [reviewing-in-chaos](./reviewing-in-chaos/)
- 对 skill 进行结构化审核，识别结构、内容、边界等问题
- 输出分级问题报告 (Critical / Important / Minor)
- 附 rubric 参考和示例

### [evaluating-in-chaos](./evaluating-in-chaos/)
- 对 skill 输出进行评测验证
- 包含方法论指南和视频混剪示例

### [lazy-in-chaos](./lazy-in-chaos/)
- 三个 skill 互驱迭代的懒人编排层
- 去仪式、留干货、补盲区（人类确认门 + 外部验证门 + 跨 session 门）
- 5 步流程 + 4 维 rubric + 3 角色轮换

ps：
在这个skill包中，我将尝试将尝试一个正在构建的想法--“忒修斯”，简单来说就是缝合，但是不是做加法，而是做思想上的根本抽取，例如在这些skill的创建中，就抽取了关于supperpower的部分内容，当这仍然是实验性质的；实际上，我们可以让现存的三个skill互相迭代（是的，我就是这么做的）；
