# in-chaos

一个在混沌中意外开启的 skill 包。每个 skill 都不知道怎么开始写的，但意外地好用。

## Skills

### [iteration-engine](./iteration-engine/)
把用户反馈转化为可复用的迭代技术体系。状态驱动的三层架构——技术（检查方法）、偏好（编排选择）、日志（原则提取）。
- 核心输出：可复用的迭代方案 (techs/)
- 状态机设计：7 状态 + transitions，任何状态都是合法入口
- 自迭代：用自身流程迭代自身

### [reviewing-skills](./reviewing-skills/)
- 对 skill 进行结构化审核，识别结构、内容、边界等问题
- 输出分级问题报告 (Critical / Important / Minor)
- 附 rubric 参考和示例

### [evaluating-skill-output](./evaluating-skill-output/)
- 对 skill 输出进行评测验证
- 包含方法论指南和视频混剪示例
