# responsibility-boundary-check
## status: active
## 覆盖维度
skill-design, module-design
## 边界
- 覆盖: 检查每个 skill/module 是否有明确的职责边界——做什么、不做什么、和谁协作
- 不覆盖: 职责内部实现质量
- 失效条件: 单体应用（只有一个模块）不需要此检查

## 验收标准
- 每个模块能清晰回答"我不做什么"
- 模块间职责无重叠
- 有明确的协作协议（谁触发谁、谁依赖谁）
## 使用说明
1. 对每个模块列出"我做什么"
2. 对每个模块列出"我不做什么"（比前者更重要）
3. 检查相邻模块的"我做什么"之间是否有重叠
4. 对每个协作关系写出协议：谁触发谁？传什么？期待什么结果？
5. 如果两个模块都在做同一件事但方式不同 → 职责重叠，需要合并或明确分界
## 盲区
- 不验证协作协议是否实际可行
- 边界可能随时间漂移——需要周期性重检
- 过于严格的边界可能阻碍必要的灵活协作
