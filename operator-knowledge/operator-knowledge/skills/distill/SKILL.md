# Distill Knowledge

中间层 skill。桥接 raw/、overlays/ 和 operators/——扫原始素材，对照已有知识，提出候选问题。

**REQUIRED:** 素材管理用 [capture](../capture/SKILL.md)，推导消化用 [derive](../derive/SKILL.md)。

## 核心理念

你不缺素材，你缺的是低负担地看见重复、矛盾和空白。distill 帮你把候选摆出来，但不替你判断哪个最重要。

## When to Use

- 用户主动问"最近有什么值得推的"、"扫一下 raw"
- 新增 raw/ 材料累积到一定量：1 篇只做单篇对照，3-4 篇可轻扫，5 篇以上适合正式 distill
- Derive 完成后，回溯 raw/ 检查是否还有关联素材

## When NOT to Use

- 用户刚投递一篇就要求完整 distill → 只做单篇对照，不抽象成稳定模式
- 只有一个知识算子且无冲突 → 不需要对照
- 用户已有明确推导目标 → 直接用 [derive](../derive/SKILL.md)

## Decision Flow

1. 用户想扫 raw/？
   - Yes → 2
   - No → 退出，或建议去 [capture](../capture/SKILL.md) 或 [derive](../derive/SKILL.md)
2. raw/ 素材数量？
   - 1 篇 → 只做单篇对照，不叫完整 distill
   - 2 篇 → 标注"素材不足"，只给轻量线索
   - 3-4 篇 → 轻扫，输出候选但降低结论强度
   - 5+ 篇 → 正式 distill
3. operators/ 有已有知识算子？
   - Yes → 执行对照阅读 + 筛选候选
   - No → 只做新概念筛选，不执行对照阅读

## 两大操作

### 对照阅读

对 raw/ 中的新素材和 operators/ 中的已有知识做并排比对：

| 对照方向 | 问题 |
|----------|------|
| **素材 → 已有知识** | "这篇日记讲的 X，你之前的知识算子 Y 能解释吗？" |
| **素材 → 知识盲区** | "这篇日记反复提到 Z，但知识库里没有——要不要推一下？" |
| **已有知识 → 素材验证** | "知识算子 A 的边界条件说'仅在 B 条件下成立'，最近素材里有反例吗？" |

### 筛选候选

扫 raw/，列出潜在可推导概念：

| 信号 | 含义 |
|------|------|
| 同一主题出现 3 次以上 | 可能值得提取 |
| 用户明确说"我不理解为什么 X" | 提醒可进入 Derive，由用户决定 |
| 和已有知识矛盾 | 触发 Evolve |
| 日记空白期后的新主题 | 可能代表认知转变 |

## 工作流

| 步骤 | 操作 |
|------|------|
| 1 | 读取 raw/ 和 overlays/，列出素材与外部标签/状态 |
| 2 | 从 operators/ 读取 index.md，拉取已有知识算子清单 |
| 3 | 逐篇新素材对照已有知识，标记：可解释 / 矛盾 / 新概念 / 无关联 |
| 4 | 输出对照报告，列出候选概念清单，由用户决定是否进入 [derive](../derive/SKILL.md) |

## 检查点 (Checkpoints)

只在用户需要选择方向时暂停。普通读取、对照、候选整理不需要确认。

| 步骤 | 检查点 | 时机 |
|------|--------|------|
| 3 | "我看到几条线索，先给你轻量版。" | 对照分析完成后 |
| 4 | "这里面哪个让你最想继续？也可以先放着。" | 输出报告后、进入 derive 前 |

不替用户做"是否进入 derive"的决定。不要用复杂表格压用户；除非用户要细看，默认给短列表。

## 输出格式

```markdown
## 对照报告（默认轻量）

- 可能相关：素材"xxxx" ↔ 知识算子"xxxx"
- 有点冲突：素材"xxxx" 里出现了旧边界没覆盖的情况
- 新候选：素材"xxxx" 反复出现主题 X
- 暂时放着：素材"xxxx" 还看不出稳定模式

你想看哪条？
```

## 输出示例（具体参照）

假设 raw/ 有 4 篇日记（学习效率低、开会走神、注意力分散、番茄工作法无效），operators/ 有"认知负荷"：

```markdown
## 对照报告

- 可能相关："学习效率低下"、"注意力分散"都能被"认知负荷"部分解释。
- 有点冲突："番茄工作法无效"不像单纯任务切换问题，可能碰到"注意力调节"。
- 新候选："注意力调节"出现得比较集中。

你想继续看"注意力调节"，还是先放着？
```

> 参照此格式和措辞风格：不替用户下结论（用"可能"、"建议"），不越权决定"必须推"。

## 异常处理

| 场景 | 处理 |
|------|------|
| raw/ 为空 | "raw/ 里还没有素材，先去投递点日记或摘录吧。" |
| operators/ 为空 | 跳过对照阅读环节，只做新概念筛选 |
| 只有 1 篇素材 | 只做单篇对照，不抽象成稳定模式 |
| 只有 2 篇素材 | 标注"素材不足"，只给轻量线索 |
| 3-4 篇素材 | 可以轻扫，但降低结论强度 |
| 所有素材均为"无关联" | 如实报告，不强行归类。建议"可能是独立主题，等更多素材出现再做判断" |
| 单一主题占据大部分素材 | 标注偏向性："你的素材高度聚焦于 X 主题，其他主题可能被遗漏" |
| 用户想在 distill 后直接写知识算子 | 建议进入 [derive](../derive/SKILL.md)，不在 distill 里横跨操作 |
| 用户表示累或不想继续 | 输出 pending 列表，不推进 derive |

## Common Mistakes

| 错误 | 后果 | 正确做法 |
|------|------|---------|
| 替用户决定"这个概念值得推" | 越权，用户才是决策者 | 列出候选，由用户选择 |
| 素材不足时强行正式 distill | 结论不稳，浪费精力 | 1 篇单篇对照，3-4 篇轻扫，5+ 正式 distill |
| 忽略无关联素材 | 漏掉潜在的独立主题 | 如实标记"无关联"，不强行归类 |
| 输出大表格和术语 | 增加使用负担 | 默认短列表，用户要细看再展开 |

## Red Flags

- "我帮你推到 derive 吧" → 只建议不替用户决定
- "这篇肯定是认知负荷的问题" → 不做确定判断，用"可能相关"
- "素材太少，就先不 distill 了" → 正常，等积累

## 合理化反驳

| 借口 | 现实 |
|------|------|
| "只有两篇素材也差不多能看出主题了" | 两篇只算线索，不能当稳定模式 |
| "这个主题我帮你直接命名吧" | 命名权在用户，你只有建议权 |
