# Capture Knowledge

文档操作 skill。管理冻结原文和外部覆盖层：保存、检索、标记、关联。不加工内容，只提供读写支撑。

**REQUIRED:** 推导消化用 [derive](../derive/SKILL.md)，概念筛选用 [distill](../distill/SKILL.md)。

## 核心理念

原始素材在变成知识之前，先是证据。capture 负责保护证据，同时把标签、状态、删除、关联这些维护信息放到旁边。

不夸讲，不批判，只记录。

## When to Use

- 用户说"记一下"、"先存着"、"放到 raw"、"归档这条"、"保存下来"
- 用户贴了一段日记、摘录、想法，需要存入 raw/
- 用户说"找一下之前关于 X 的记录"、"raw 里有哪些关于 Y 的"
- raw/ 文件多了，需要清理、标记状态或建立关联

## 三大操作

### Capture（投递）

| 触发 | 动作 |
|------|------|
| "记一下"、"先存着" | 写入 raw/ 原文；把标题、来源、标签写到覆盖层 |
| 用户贴了一段日记/摘录/想法 | 写入 raw/，不改写、不加摘要、不加评论 |

文件命名：`raw/{YYYY-MM-DD}-{标题}.md`

### 覆盖层规范

raw 文件不写 YAML 头，不追加状态。元数据写到 `overlays/raw-index.md` 或等价索引：

```markdown
| id | path | date | source | tags | status | note |
|----|------|------|--------|------|--------|------|
| 2026-05-09-抗拒做计划 | raw/2026-05-09-抗拒做计划.md | 2026-05-09 | 日记 | 计划, 动机 | active | 原文冻结 |
```

如需记录修正、删除、关联，写到 `overlays/raw-changes.md` 或 `overlays/raw-links.md`，不要改 raw 原文。

### Retrieve（检索）

| 触发 | 动作 |
|------|------|
| "找一下之前关于 X 的记录" | 在 raw/ 中全文或标签搜索 |
| "raw 里有哪些关于 Y 的？" | 列出匹配文件及摘要 |

### Clean（清理）

不物理删除，不改 raw 文件，只在覆盖层标记。

- 废弃文件：在覆盖层设 `status: archived`
- 被 distill 选中进入 derive 的材料：在覆盖层设 `status: distilling`
- 发现标题或标签不准：更新覆盖层，不改文件名，除非用户明确要求重命名

## 工作流

| 步骤 | 操作 |
|------|------|
| 1 | 用户投递内容 → 生成文件名 + 旁路记录 |
| 2 | 写入 raw/ 原文，必要时创建 `overlays/` |
| 3 | 用户检索时 → 全文搜索 + 覆盖层过滤，返回少量匹配项 |
| 4 | distill 请求候选材料时 → 提供 raw/ 文件列表和覆盖层信息 |

## 检查点 (Checkpoints)

只在会改变原文归档或用户选择成本较高时停下。不要把普通检索做成审批流程。

| 工作流步骤 | 检查点 | 时机 |
|-----------|--------|------|
| 1. 写入原文前 | "我会把它当原文存起来，不改内容；标题先用'{标题}'。" | 内容和标题不确定时 |
| 2. 大量检索结果 | "找到 N 条，我先给最相关的几条。" | 结果太多，继续展开会增加负担时 |

默认轻量执行；用户不满意标题、标签、展示深度时再调整。

## When NOT to Use

- 用户已经有一个明确概念想消化 → [derive](../derive/SKILL.md)
- 用户想知道 raw/ 里有没有值得推导的 → [distill](../distill/SKILL.md)
- 用户只是想聊天记东西 → 直接对话即可

## Common Mistakes

| 错误 | 后果 | 正确做法 |
|------|------|---------|
| 投递时帮用户加工内容 | 污染原始素材 | 只记录原文，摘要写覆盖层 |
| 把 metadata 写进 raw | 原文不再冻结 | metadata 写 `overlays/` |
| 物理删除文件 | 素材丢失不可回溯 | 覆盖层标 `archived` |

## Red Flags

- "这段我帮你总结一下再存" → 不要加工，直接存原文
- "这个太乱了，先整理好再放进 raw" → 乱是原始状态，先存后整
- "我顺手在文件头加个 tags" → 不要写进 raw，写覆盖层
- "以后也不会看了，删掉吧" → 逻辑标记，不物理删除

## 合理化反驳

| 借口 | 现实 |
|------|------|
| "我就帮你稍微精简一下再存" | 精简 = 编辑 = 污染原始素材 |
| "元数据写文件头更方便" | 方便会污染证据层；覆盖层才可演化 |
| "这个文件以后不会用了，直接删" | 未来可能成为对照阅读的参照物 |

## 异常处理

| 场景 | 处理 |
|------|------|
| raw/ 目录不存在 | 自动创建 `raw/` 目录；需要元数据时创建 `overlays/` |
| 用户投递超长内容（>5000字） | 不做截断，完整存入；提示"内容较长，建议加分段标题" |
| 文件名冲突 | 加序号后缀 raw/2026-05-09-日记-2.md |
| 用户说"存下来"但没给内容 | 追问："标题和内容是什么？" |
| 检索无结果 | 提示扩大关键词、检查时间范围或换源标注 |
| 用户想存非文本内容（图片/链接） | 存链接或描述替代，原文注明格式；不存储二进制 |
| 元数据缺少标签 | 先给 1-3 个轻量标签写覆盖层；用户可随时改 |
| raw/ 文件数量超过 50 | 提示"素材较多，建议做一次 distill" |
